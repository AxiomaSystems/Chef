# Changes - May 7, 2026

This document summarizes the inventory quantity and cart deduction work completed on May 7, 2026.

## Inventory Quantity Editing

**Change**: Inventory items now carry visible quantity and unit values instead of only displaying `In stock`.

**Details**:
- Added quantity and unit inputs when adding ingredients from the inventory `Add` modal.
- Added editable quantity and unit fields on existing inventory rows.
- Added a server action for updating inventory item quantities from the web app.
- Added a backend `PATCH /me/kitchen-inventory/:id` endpoint for inventory quantity updates.

**Files Changed**:
- `apps/web/src/app/inventory/inventory-client.tsx`
- `apps/web/src/app/inventory/actions.ts`
- `apps/api/src/ingredients/ingredients.controller.ts`
- `apps/api/src/ingredients/ingredients.service.ts`
- `apps/api/src/ingredients/dto/update-kitchen-inventory-item.dto.ts`

## Cart Inventory Deduction

**Change**: Cart ingredient aggregation now includes inventory-aware quantity fields and uses remaining quantity for shopping matches.

**Details**:
- Extended `AggregatedIngredient` with:
  - `inventory_amount`
  - `inventory_unit`
  - `remaining_to_buy`
  - `deduction_possible`
- Shopping cart creation now deducts matching inventory quantities before product matching.
- If inventory fully covers an ingredient, the remaining amount to buy is `0`.
- If units are compatible, the cart buys only the remaining amount.
- Basic conversions were added for common volume and weight units, including `ml`, `l`, `cup`, `tsp`, `tbsp`, `g`, `kg`, `oz`, and `lb`.

**Files Changed**:
- `packages/shared/src/aggregation.ts`
- `apps/api/src/cart/cart.service.ts`
- `apps/api/src/common/http/swagger.dto.ts`
- `apps/api/src/cart/cart.service.spec.ts`
