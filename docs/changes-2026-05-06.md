# Changes - May 6, 2026

This document summarizes the fixes and improvements made to the Chef codebase on May 6, 2026.

## Build Fixes

### 1. API Build - Prisma Client Regeneration

**Issue**: The API build was failing with 13 TypeScript errors related to missing Prisma models and enums:
- `UserMemoryConfidence` and `UserMemorySource` enums
- `UserFoodRule`, `UserGoal`, `UserPantryStaple` models and their payload types
- Missing Prisma client methods for the new models

**Root Cause**: The Prisma client wasn't regenerated after the schema was updated with new user profile memory models.

**Fix**: Regenerated the Prisma client using `pnpm prisma generate` in the API directory.

**Files Changed**:
- `apps/api/generated/prisma/index.d.ts` (auto-generated)
- `apps/api/generated/prisma/index.js` (auto-generated)

### 2. Web Build - Missing Handler Function

**Issue**: The web build was failing with a TypeScript error:
```
Type error: Cannot find name 'handleAdded'.
```

**Root Cause**: The `VisionScanModal`, `CameraModal`, and `AddItemModal` components were calling a `handleAdded` function that wasn't defined in the `InventoryClient` component.

**Fix**: Added the missing `handleAdded` function to handle items added through the modal components.

**Files Changed**:
- `apps/web/src/app/inventory/inventory-client.tsx`

**Code Added**:
```typescript
function handleAdded(item: KitchenInventoryItem) {
  setItems((prev) => [realToDisplay(item), ...prev]);
}
```

## Infrastructure Setup

### 3. Vision Sidecar Service

**Issue**: Vision sidecar was unavailable at http://localhost:8000, preventing real YOLO scans.

**Fix**: Set up and started the FastAPI vision service:
1. Ran `pnpm vision:setup` to create Python virtual environment and install dependencies
2. Started the FastAPI server using `apps/vision-lab/run_fastapi.ps1`
3. Verified the service is running and accessible

**Service Details**:
- **URL**: http://localhost:8000
- **Technology**: FastAPI with Uvicorn
- **Purpose**: Handles YOLO object detection and ingredient classification
- **Endpoints**: `/detect/media` for image/video processing, `/docs` for API documentation

**Files Changed**:
