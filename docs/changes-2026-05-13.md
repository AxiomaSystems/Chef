# Changes - May 13, 2026

This document summarizes the kitchen inventory defaults and classification updates completed on May 13, 2026.

## Inventory Defaults

**Change**: Inventory rows now show sensible default quantities and units instead of blank quantity controls or generic `In stock` text.

**Details**:

- The ingredient picker pre-fills a default quantity and unit for every catalog ingredient.
- Existing inventory rows derive display defaults from the saved inventory item, linked ingredient `default_unit`, or a local name-based fallback.
- Default quantities remain lightweight estimates for user editing, not exact pantry accounting.

**Files Changed**:

- `apps/web/src/app/inventory/inventory-client.tsx`
- `apps/api/src/ingredients/ingredients.service.ts`
- `apps/api/prisma/seed/ingredients.js`

## Unit Editing

**Change**: Inventory unit editing now uses dropdown controls.

**Details**:

- The add-ingredient modal uses a unit dropdown instead of a free-text unit input.
- Existing inventory rows use the same unit option set.
- Changing a unit on an existing row saves immediately; quantity edits continue saving on blur.

## Ingredient Classification

**Change**: Ingredient categories now use broader classification rules.

**Details**:

- Backend-created ingredients now infer `category` and `defaultUnit` when created by `canonical_name`.
- Seeded ingredient category inference covers common proteins, dairy/eggs, fruits, produce, grains, pantry staples, and spices.
- The inventory page also applies a frontend fallback so older rows categorized as `other` can still display in a better group before the database is reseeded.

## Verification

Validated with:

1. `pnpm --filter api test -- ingredients.service.spec.ts --runInBand`
2. `pnpm --filter api build`
3. `pnpm --filter web build`
